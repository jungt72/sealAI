## Router Conditions (state-dependent)
- `_resume_router`: uses `awaiting_user_confirmation` + `confirm_decision` → resume/reject; uses `awaiting_user_input` → ingest_missing; else frontdoor. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:433-439`.
- `_frontdoor_router`: uses `intent.goal` to pick `finalize` for smalltalk/out_of_scope, else discovery. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:87-92`.
- `_discovery_intake_router`: routes to `ask_missing` if `awaiting_user_input` or `ask_missing_request`. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:346-349`.
- `_discovery_router`: `ask_missing` if awaiting/ask_missing_request; `finalize` for smalltalk/out_of_scope; else `curate`. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:352-359`.
- `_curator_router`: `cluster` if `intent.goal == design_recommendation` else `supervisor`. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:361-365`.
- `_policy_gate_router`: `ask_missing` if awaiting/ask_missing_request else `finalize`. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:368-371`.
- `_supervisor_policy_router`: maps `next_action`/`pending_action` to route labels (calc/design/product/knowledge/finalize). Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:397-430`.
- `_knowledge_target_router`: uses `intent.key` / `knowledge_type` to choose `knowledge_material_node` / `knowledge_lifetime_node` / `generic_sealing_qa_node`. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:488-495`.
- `_confirm_checkpoint_router`: `awaiting_user_confirmation` → `confirm_checkpoint_node` else delegate to `_supervisor_policy_router`. Evidence: `backend/app/langgraph_v2/sealai_graph_v2.py:442-445`.
