# v1.3 Patch C1: Knowledge Depth Expansion & Source Grounding

## 1. Summary of Changes
Patch C1 strengthens the technical knowledge base by introducing structured, source-grounded facts for engineering materials. It bridges the gap between raw RAG chunks and the UI-facing workspace by extracting technical properties directly from authoritative documents.

## 2. Key Technical Upgrades
- **Structured Grounding**: Added `grounded_facts` to `CandidateItem` (State) and `MaterialFitItem` (Projection).
- **Automated Extraction**: Enhanced `node_prepare_contract.py` with logic to scan RAG metadata and text for technical properties (e.g., Temperature Range, Shore Hardness) relevant to selected materials.
- **Source Transparency**: Each grounded fact now carries an explicit document source reference, visible in the UI.
- **Enhanced UI**: The `PartnerMatchingPanel` now displays these technical details within the material cards, providing immediate justification for why a material is considered viable.

## 3. Grounding Depth
The system now extracts and surfaces:
- **Temperature Limits**: Min/Max values grounded in material datasheets.
- **Material Hardness**: Shore A values derived from normative references or technical specs.
- **Source IDs**: Exact pointers to the evidence chunks used.

## 4. Continuity & Governance
- Existing workflow gates (Confirmation, Selection, Handover) remain fully intact.
- Grounded facts are tied to the Assertion Cycle, ensuring they remain fresh or mark the state as `stale` if parameters change.

## 5. Next Steps
- **C2: Semantic Claim Mapping**: Deeper mapping of natural language claims to structured conflicts.
- **C3: Dynamic Specificity Elevation**: Allowing the system to automatically request specific missing data to move from family-level to compound-level recommendations.
