# v1.3 Patch C3: Guided Specificity Elevation

## 1. Summary of Changes
Patch C3 introduces a "Guided Specificity Elevation" path in the Case Workspace. It helps users understand why the system might only be providing family-level material recommendations and exactly which information is needed to reach compound-level specificity.

## 2. Key Technical Upgrades
- **Elevation Detection**: The projection logic in `case_workspace.py` now detects "Specificity Gaps" by analyzing the difference between required specificity and available material candidates.
- **Guided Hints**: Surfaced explicit, actionable hints in the UI when elevation is possible.
- **Schema Expansion**: Added `elevation_possible`, `elevation_hints`, and `elevation_target` to `SpecificityInfo`.
- **UI Integration**: Added a new "Increase Specificity" section in the `CaseStatusPanel`, visually highlighting the path to a higher-quality technical result.

## 3. Elevation Logic (Patch C3)
Elevation hints are triggered in two main scenarios:
1. **Data Gaps**: Critical technical parameters (Medium, Pressure, Temperature) are missing, preventing a compound-specific rating.
2. **Material Genericity**: Only general material families (e.g., NBR, FKM) are identified, but no specific compound or product name has been provided or extracted.

## 4. UI Visualization
- **Target Badge**: Clearly shows the elevation target (e.g., "Target: Compound").
- **Actionable List**: Points out the specific missing fields or information required.
- **Contextual Placement**: Located right under the Completeness section for maximum relevance.

## 5. Verification Results
- **Unit Tests**: 3 new test cases verified detection of missing parameters and family-level candidates.
- **Regression**: All 75 existing workspace projection tests passed.

## 6. Open Topics
- **Interactive Prompts**: Future iterations (C4) could allow clicking these hints to immediately focus the chat or trigger a specific clarification tool.
- **Specificity Escalation Node**: An active agent node that proactively asks for these parameters instead of just surfacing them in the sidebar.
