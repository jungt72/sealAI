# v1.3 Patch C4: Specificity Action Bridge

## 1. Summary of Changes
Patch C4 transforms the passive specificity hints from C3 into a structured "Action Bridge". It allows the user to immediately act on specificity gaps by clicking on hints, which pre-populates the chat with relevant technical prompts.

## 2. Key Technical Upgrades
- **Structured Elevation Hints**: Replaced simple strings with a `ElevationHint` model containing `label`, `field_key`, `reason`, `priority`, and `action_type`.
- **Action Bridge Logic**: Implemented `onActionBridge` in `ChatInterface` to connect sidebar hints with the `ChatComposer`.
- **Interactive UI**: The `CaseStatusPanel` now renders hints as clickable buttons that guide the user's next chat interaction.
- **Prioritization**: Hints are now sorted by technical priority (e.g., Medium/Pressure/Temperature gaps prioritized over generic material names).

## 3. Action Bridge Mechanics
- **Data Gaps**: Clicking a missing parameter hint (e.g., "Define Medium") generates a prompt like: *"Lass uns über Medium sprechen, um die Spezifität zu erhöhen."*
- **Material Gaps**: Clicking "Specify exact compound" generates: *"Ich möchte das Material genauer spezifizieren (z.B. genaue Compound-Bezeichnung)."*

## 4. Continuity & Governance
- The "Next Best Action" high-level workflow remains unchanged.
- Specificity hints act as a sub-workflow within the "Discovery" or "Analysis" phase.
- All 75 existing workspace projection tests and 3 new specificity tests passed.

## 5. Next Steps
- **C5: Proactive Specificity Node**: An active agent node that can detect these gaps and proactively ask the user during the chat turn, rather than waiting for a sidebar click.
- **Automated Field Selection**: Expanding the bridge to automatically open specific UI tool overlays or parameter editors.
