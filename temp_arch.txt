## 3. RAG-Pipeline: Paperless-ngx → Qdrant

Das ist der wichtigste Datenfluss neben dem Chat.
Admin pflegt die Wissensbasis — SeaLAI nutzt sie im Governed-Pfad.

### 3.1 Konzept

```
ADMIN                  PAPERLESS-NGX           TIKA
  │                         │                   │
  │ lädt PDF hoch           │                   │
  │────────────────────────►│                   │
  │                         │ OCR-Job           │
  │                         │──────────────────►│
  │                         │◄─────────────────-│
  │                         │  extrahierter Text │
  │ vergibt Tags            │                   │
  │────────────────────────►│                   │
  │                         │                   │
  │                         │ Webhook →         │
  │                         │ Ingest-Service    │
  │                         │──────────────────►│
  │                         │               BACKEND
  │                         │          Chunking + Embedding
  │                         │          + Qdrant Write
```

### 3.2 Tagging-Schema in Paperless-ngx

Der Admin vergibt beim Upload folgende Tags.
Diese Tags werden direkt zu Qdrant-Metadaten.

**Pflicht-Tags (mindestens einer):**

| Tag-Präfix | Beispiele | Bedeutung |
|---|---|---|
| `sts_mat:` | `sts_mat:STS-MAT-SIC-A1` | Zugehöriges Material |
| `sts_type:` | `sts_type:STS-TYPE-GS-CART` | Zugehöriger Dichtungstyp |
| `doc_type:` | `doc_type:datasheet` | Art des Dokuments |
| `lang:` | `lang:de` oder `lang:en` | Sprache |

**Optionale Tags:**

| Tag-Präfix | Beispiele | Bedeutung |
|---|---|---|
| `sts_med:` | `sts_med:STS-MED-NACL-B` | Zugehöriges Medium |
| `sts_rs:` | `sts_rs:STS-RS-CHEM-B1` | Zugehörige Requirement Class |
| `source:` | `source:freudenberg` | Hersteller/Quelle |
| `norm:` | `norm:DIN-EN-12756` | Norm-Referenz |
| `industry:` | `industry:chemie` | Branche |

**doc_type Werte:**
```
datasheet        → Technisches Datenblatt eines Materials/Produkts
norm_excerpt     → Normauszug (DIN, ISO, API)
failure_mode     → Schadensbild-Dokumentation
application_note → Anwendungshinweis
calculation_basis→ Berechnungsgrundlage
manufacturer_cap → Herstellerfähigkeits-Info
```

### 3.3 Ingestion Service (Backend)

```python
# backend/app/rag/ingest_service.py

from paperless_client import PaperlessClient
from qdrant_client import QdrantClient
from sentence_transformers import SentenceTransformer
import asyncio

class RAGIngestService:
    """
    Hört auf Paperless-Webhooks und ingestiert neue/geänderte
    Dokumente in Qdrant.
    Läuft als Background-Worker, nicht im kritischen Chat-Pfad.
    """

    def __init__(self):
        self.paperless = PaperlessClient(
            base_url="http://paperless:8000",
            token=settings.PAPERLESS_API_TOKEN
        )
        self.qdrant = QdrantClient(host="qdrant", port=6333)
        self.embedder = SentenceTransformer(
            "sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2"
        )

    async def handle_webhook(self, document_id: int):
        """Paperless-Webhook → Qdrant"""

        # 1. Dokument + Tags von Paperless holen
        doc = await self.paperless.get_document(document_id)
        tags = await self.paperless.get_tags_for_document(document_id)
        content = doc.content  # bereits OCR'd von Tika via Paperless

        # 2. Tags in Metadaten-Dict übersetzen
        metadata = self._parse_tags(tags)

        # Pflicht-Tag fehlt → Log + Skip
        if not metadata.get("doc_type"):
            logger.warning(f"Doc {document_id} fehlt doc_type-Tag — übersprungen")
            return

        # 3. Chunking (Strategie je nach doc_type)
        chunks = self._chunk(content, metadata["doc_type"])

        # 4. Embedding
        embeddings = self.embedder.encode(
            [c.text for c in chunks],
            batch_size=32,
            show_progress_bar=False
        )

        # 5. Qdrant-Upsert (idempotent — doc_id als Point-ID-Basis)
        points = [
            models.PointStruct(
                id=f"{document_id}_{i}",
                vector={"dense": emb.tolist()},
                payload={
                    "text": chunk.text,
                    "doc_id": document_id,
                    "doc_title": doc.title,
                    "doc_version_hash": doc.checksum,
                    "page": chunk.page,
                    **metadata
                }
            )
            for i, (chunk, emb) in enumerate(zip(chunks, embeddings))
        ]

        await self.qdrant.upsert(
            collection_name="sealai_technical_docs",
            points=points
        )

        logger.info(
            f"Ingestiert: {doc.title} → {len(points)} Chunks → Qdrant"
        )

    def _parse_tags(self, tags: list[str]) -> dict:
        """Tags → Metadaten-Dict"""
        meta = {
            "sts_mat_codes": [],
            "sts_type_codes": [],
            "sts_med_codes": [],
            "sts_rs_codes": [],
            "doc_type": None,
            "language": "de",
            "source": None,
            "norm_references": [],
            "industries": [],
        }
        for tag in tags:
            if tag.startswith("sts_mat:"):
                meta["sts_mat_codes"].append(tag[8:])
            elif tag.startswith("sts_type:"):
                meta["sts_type_codes"].append(tag[9:])
            elif tag.startswith("sts_med:"):
                meta["sts_med_codes"].append(tag[8:])
            elif tag.startswith("sts_rs:"):
                meta["sts_rs_codes"].append(tag[7:])
            elif tag.startswith("doc_type:"):
                meta["doc_type"] = tag[9:]
            elif tag.startswith("lang:"):
                meta["language"] = tag[5:]
            elif tag.startswith("source:"):
                meta["source"] = tag[7:]
            elif tag.startswith("norm:"):
                meta["norm_references"].append(tag[5:])
            elif tag.startswith("industry:"):
                meta["industries"].append(tag[9:])
        return meta

    def _chunk(self, text: str, doc_type: str) -> list[Chunk]:
        """
        Chunking-Strategie je nach Dokumenttyp.
        Datasheets: 512 Token, 50 Overlap
        Normauszüge: 256 Token, Section-aware
        Schadensbilder: whole document (kurz genug)
        """
        if doc_type == "failure_mode":
            return [Chunk(text=text, page=0)]
        elif doc_type == "norm_excerpt":
            return self._section_aware_chunk(text, max_tokens=256, overlap=30)
        else:
            return self._sliding_window_chunk(text, max_tokens=512, overlap=50)
```

## 4. EvidenceQuery → Qdrant (RAG-Retrieval)

So fragt der Governed-Subgraph die Wissensbasis ab:

```python
# backend/app/evidence/retrieval.py

async def retrieve_evidence(query: EvidenceQuery) -> list[EvidenceItem]:
    """
    Hybrid Retrieval: Dense (Semantic) + Sparse (BM25)
    Gefiltert nach STS-Codes aus dem aktuellen Case.
    """

    # Embedding der Query
    dense_vector = embedder.encode(query.topic).tolist()

    # BM25-Encoding
    sparse_vector = bm25_encoder.encode(query.topic)

    # Filter: nur Dokumente die zu den erkannten STS-Codes passen
    sts_filter = None
    if query.detected_sts_codes:
        sts_filter = models.Filter(
            should=[
                models.FieldCondition(
                    key="sts_mat_codes",
                    match=models.MatchAny(any=query.detected_sts_codes)
                ),
                models.FieldCondition(
                    key="sts_type_codes",
                    match=models.MatchAny(any=query.detected_sts_codes)
                ),
            ]
        )

    # Hybrid Query
    results = await client.query_points(
        collection_name="sealai_technical_docs",
        prefetch=[
            # Dense: semantische Ähnlichkeit
            models.Prefetch(
                query=dense_vector,
                using="dense",
                limit=20
            ),
            # Sparse: BM25 Keyword-Match
            models.Prefetch(
                query=models.SparseVector(**sparse_vector),
                using="sparse",
                limit=20
            ),
        ],
        # RRF Fusion: kombiniert beide Rankings
        query=models.FusionQuery(fusion=models.Fusion.RRF),
        query_filter=sts_filter,
        with_payload=True,
        limit=query.max_results,
    )

    return [
        EvidenceItem(
            text=r.payload["text"],
            source=r.payload.get("source"),
            doc_title=r.payload.get("doc_title"),
            doc_version_hash=r.payload.get("doc_version_hash"),
            norm_references=r.payload.get("norm_references", []),
            trust_level=_calc_trust_level(r.payload),
            score=r.score
        )
        for r in results.points
    ]
```

## 5. Gotenberg — PDF-Erzeugung

Das Inquiry-Dokument wird als HTML gerendert und via Gotenberg zu PDF konvertiert.

```python
# backend/app/documents/pdf_generator.py

import httpx
from jinja2 import Environment, FileSystemLoader

PDF_TEMPLATE_DIR = "backend/app/templates/pdf/"

async def generate_inquiry_pdf(state: SealAIState) -> bytes:
    """
    1. Jinja2-Template rendern (HTML)
    2. HTML → PDF via Gotenberg
    3. Idempotency-Check via Redis
    """

    # Idempotency: bereits erzeugt?
    key = state.action_readiness.idempotency_key
    cached = await redis.get(f"pdf:{key}")
    if cached:
        return cached

    # HTML aus Jinja2-Template
    env = Environment(loader=FileSystemLoader(PDF_TEMPLATE_DIR))
    template = env.get_template("inquiry.html.j2")
    html = template.render(
        case_number=state.case_id,
        parameters=state.normalized.canonical_parameters,
        parameter_status=state.normalized.parameter_status,
        calculations=build_calculations_display(state.derived),
        preselection=state.decision.preselection,
        assumptions=state.decision.assumptions,
        open_points=state.decision.open_points,
        applicable_norms=state.derived.applicable_norms,
        disclaimer=INQUIRY_DISCLAIMER,
        created_at=datetime.utcnow().isoformat(),
        basis_hash=state.decision.decision_basis_hash,
    )

    # Gotenberg: HTML → PDF
    async with httpx.AsyncClient() as client:
        response = await client.post(
            "http://gotenberg:3000/forms/chromium/convert/html",
            files={"files": ("index.html", html, "text/html")},
            data={
                "marginTop": "1",
                "marginBottom": "1",
                "marginLeft": "1",
                "marginRight": "1",
                "paperWidth": "8.27",   # A4
                "paperHeight": "11.69",
            },
            timeout=30.0
        )
        pdf_bytes = response.content

    # Cache + Audit
    await redis.setex(f"pdf:{key}", 86400, pdf_bytes)
    await write_pdf_audit(state, key)

    return pdf_bytes
```

## 7. LangGraph + Redis: Session-Management

```python
# backend/app/runtime/session_manager.py

from langgraph.checkpoint.redis.aio import AsyncRedisSaver

class SessionManager:

    def __init__(self):
        self.checkpointer = AsyncRedisSaver.from_conn_string(
            redis_url=settings.REDIS_URL,
            ttl={"default": 86400 * 7}  # 7 Tage
        )
        self.graph = build_graph().compile(
            checkpointer=self.checkpointer
        )

    def build_config(self, case_id: str, session_id: str) -> dict:
        return {
            "configurable": {
                "thread_id": f"sealai:v3:{case_id}:{session_id}",
                "checkpoint_ns": "v3"
            }
        }

    async def resume_session(self, case_id: str, session_id: str) -> SessionInfo:
        """Prüft ob ein aktiver Checkpoint existiert."""
        config = self.build_config(case_id, session_id)
        checkpoint = await self.checkpointer.aget(config)

        if checkpoint:
            state = checkpoint["channel_values"]
            return SessionInfo(
                is_resumed=True,
                last_outward_class=state.get("decision", {}).get("outward_class"),
                summary=build_resume_summary(state),
                case_revision=state.get("case_revision", 0)
            )
        return SessionInfo(is_resumed=False)

    async def migrate_v2_to_v3(self, case_id: str, session_id: str):
        """Alte v2-Checkpoints werden nicht migriert — sauberer Schnitt."""
        v2_pattern = f"sealai:v2:{case_id}:*"
        old_keys = await redis.keys(v2_pattern)
        if old_keys:
            await redis.delete(*old_keys)
            logger.info(f"Session v2→v3 migriert (neu gestartet): {session_id}")
```

## 10. Streaming-Architektur (SSE)

```python
# backend/app/api/handlers/chat_handler.py

from fastapi import APIRouter
from fastapi.responses import StreamingResponse

router = APIRouter()

@router.post("/agent/chat/stream")
async def chat_stream(
    request: ChatRequest,
    user: SeaLAIUser = Depends(get_current_user)
):
    async def generate():
        config = session_manager.build_config(
            request.case_id,
            request.session_id
        )

        async for event in graph.astream(
            input=build_graph_input(request),
            config=config,
            stream_mode=["custom", "messages"]
        ):
            event_type, payload = event

            # Phase 1: Progress-Events (kein Text)
            if event_type == "custom":
                yield f"data: {json.dumps({'type': 'progress', **payload})}\n\n"

            # Phase 2: Token-Stream (nur renderer_node)
            elif event_type == "messages":
                chunk, metadata = payload
                if metadata.get("langgraph_node") == "renderer_node":
                    if hasattr(chunk, "content") and chunk.content:
                        yield f"data: {json.dumps({'type': 'token', 'content': chunk.content})}\n\n"

        yield "data: [DONE]\n\n"

    return StreamingResponse(
        generate(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no"  # nginx: kein Buffering bei SSE
        }
    )
```

## 11. nginx-Konfiguration (SSE-kritisch)

SSE braucht spezielle nginx-Config — sonst buffert nginx und der Stream kommt nicht durch.

```nginx
# /etc/nginx/conf.d/sealai.conf

upstream backend {
    server backend:8000;
}

upstream frontend {
    server frontend:3000;
}

server {
    listen 443 ssl;
    server_name sealai.example.com;

    # Frontend
    location / {
        proxy_pass http://frontend;
        proxy_set_header Host $host;
    }

    # Backend API
    location /api/ {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # SSE: kein Buffering!
    location /api/agent/chat/stream {
        proxy_pass http://backend;
        proxy_set_header Host $host;

        # SSE-kritisch:
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
        proxy_set_header Connection '';
        proxy_http_version 1.1;
        chunked_transfer_encoding on;

        # CORS für EventSource
        add_header Access-Control-Allow-Origin $http_origin always;
        add_header Cache-Control no-cache always;
    }

    # Paperless (nur für Admin-Rolle erreichbar)
    location /admin/paperless/ {
        proxy_pass http://paperless:8000/;
        # Keycloak-Auth prüfen
        auth_request /auth/verify-admin;
    }
}
```
